## Contributing

To add or update a translation, edit the files under `assets/bundles/` (app UI) and `assets/docs/` (documents content). The CI regenerates `docs/i18n.md` and all READMEs automatically when changes are detected.

Each bundle file must include 4 metadata fields at the top:

```jsonc
// assets/bundles/en.json
{
    "lang": "English",              // language name of that lang
    "langCode": "en",               // ISO 639-1 language code
    "langRegion": "United States",  // script / region name
    "langRegionCode": ["US", "GB"], // [script code(optional), region codes...]

    // ... translations below
}
```

- **`lang`**: Language name, in the language using(e.g. `中文` for `zh`)
- **`langCode`**: [ISO 639-1](https://www.qiuwenbaike.cn/wiki/ISO_639-1) language code
- **`langRegion`**: Can be either a script name (e.g. `"简体"`, `"繁體"` for `zh`) or region name (e.g. `中国大陆` for `zh`), script name is preferred
- **`langRegionCode`**: First element should be the [ISO 15924](https://unicode.org/iso15924/iso15924-codes.html) alpha-4 script code (optional), followed by [ISO 3166-1](https://www.qiuwenbaike.cn/wiki/ISO_3166-1) alpha-2 region codes (1 or more)

### Add a new language

1. Copy `assets/bundles/en.json` → `assets/bundles/{code}.json`
2. Copy `assets/docs/en.json` → `assets/docs/{code}.json`
3. Translate the values
4. Submit a Pull Request

### Modify existing translations

- **App UI**: Edit the corresponding file in `assets/bundles/`
- **Documents**: Edit the corresponding file in `assets/docs/`
- **Templates**: Edit the files in `assets/templates/` - this may break document translations, check them before modifing.
